# 预训练语言模型

## 什么是预训练语言模型？

预训练语言模型（Pre-trained Language Models, PLMs）是指在大规模无标注文本数据上进行预训练，学习通用语言表示的神经网络模型。这些模型可以迁移到各种下游 NLP 任务中，通过微调（Fine-tuning）或提示学习（Prompt Learning）获得优异性能。

## 预训练范式

### 1. 自监督学习
利用文本自身的结构创建监督信号，无需人工标注：
- **语言建模**：预测下一个词
- **掩码语言建模**：预测被掩码的词
- **句子顺序预测**：判断句子顺序是否正确

### 2. 预训练-微调范式
```
大规模无标注数据预训练 → 下游任务微调
```

## 代表性模型

### 1. ELMo（Embeddings from Language Models）
- **提出时间**：2018年
- **核心思想**：基于双向 LSTM 的上下文相关词嵌入
- **特点**：
  - 动态词表示（一词多义）
  - 深层双向表示
  - 字符级 CNN 处理词形

### 2. GPT 系列

#### GPT-1
- **提出时间**：2018年
- **架构**：Transformer 解码器
- **预训练任务**：单向语言建模
- **参数量**：1.17亿

#### GPT-2
- **提出时间**：2019年
- **参数量**：15亿
- **特点**：Zero-shot 能力显著提升

#### GPT-3
- **提出时间**：2020年
- **参数量**：1750亿
- **特点**：
  - 强大的 Few-shot 学习能力
  - 涌现能力（Emergent Abilities）
  - 能完成多种任务而无需微调

#### GPT-4
- **提出时间**：2023年
- **特点**：
  - 多模态能力（文本+图像）
  - 更强的推理能力
  - 更长的上下文窗口

### 3. BERT 系列

#### BERT
- **提出时间**：2018年
- **架构**：Transformer 编码器
- **预训练任务**：
  - MLM（Masked Language Model）：随机掩码 15% 的词进行预测
  - NSP（Next Sentence Prediction）：判断两个句子是否连续
- **参数量**：1.1亿（Base）、3.4亿（Large）

#### RoBERTa
- **改进**：
  - 移除 NSP 任务
  - 更大的 batch size 和训练数据
  - 动态掩码策略

#### ALBERT
- **改进**：
  - 参数共享减少参数量
  - 因子化嵌入参数化
  - 句子顺序预测替代 NSP

### 4. 中文预训练模型

#### 中文 BERT
- **BERT-wwm**：全词掩码 BERT
- **MacBERT**：用相似词替换掩码词

#### ERNIE 系列（百度）
- **ERNIE 1.0**：实体级掩码
- **ERNIE 2.0**：持续学习框架
- **ERNIE 3.0**：知识增强

#### 通义千问（阿里云）
- Qwen 系列模型
- 支持中英文
- 开源可商用

#### ChatGLM（智谱AI）
- 中英双语对话模型
- 针对中文优化

## 预训练模型的应用

### 1. 文本分类
```python
from transformers import BertForSequenceClassification
model = BertForSequenceClassification.from_pretrained('bert-base-chinese', num_labels=2)
```

### 2. 命名实体识别
```python
from transformers import BertForTokenClassification
model = BertForTokenClassification.from_pretrained('bert-base-chinese', num_labels=9)
```

### 3. 问答系统
```python
from transformers import BertForQuestionAnswering
model = BertForQuestionAnswering.from_pretrained('bert-base-chinese')
```

### 4. 文本生成
使用 GPT 系列模型进行文本生成、摘要、翻译等任务。

## 微调策略

### 1. 全量微调（Full Fine-tuning）
更新模型的所有参数，适合数据充足的场景。

### 2. 部分微调
- **冻结底层**：只微调顶层参数
- **逐层解冻**：逐步解冻更多层

### 3. 参数高效微调（PEFT）
- **Adapter**：在 Transformer 层间插入小型适配器
- **LoRA**：低秩适配，只训练低秩矩阵
- **Prompt Tuning**：优化提示嵌入
- **Prefix Tuning**：优化前缀参数

## 未来趋势

4. **高效推理**：模型压缩、量化、蒸馏
