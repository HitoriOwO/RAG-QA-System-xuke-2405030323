# 词嵌入技术详解

## 什么是词嵌入？

词嵌入（Word Embedding）是将词汇映射到低维连续向量空间的技术。通过词嵌入，语义相似的词在向量空间中距离较近，计算机可以更好地理解和处理自然语言。

## 传统表示方法

### 1. One-Hot 编码
将每个词表示为长度为词汇表大小的向量，只有一个位置为 1，其余为 0。

**示例：**
```
"猫" = [1, 0, 0, 0, 0]
"狗" = [0, 1, 0, 0, 0]
"鱼" = [0, 0, 1, 0, 0]
```

**缺点：**
- 维度灾难（词汇表通常数万到数十万）
- 无法表示词之间的语义关系
- 稀疏表示，存储效率低

### 2. 词袋模型（Bag of Words）
忽略词序，将文本表示为词的集合。

**缺点：**
- 丢失词序信息
- 无法捕捉语义关系

## 分布式表示

### 核心思想
"一个词的含义由它周围的词决定" —— 分布假说（Distributional Hypothesis）

### Word2Vec（2013）
Google 提出的经典词嵌入模型，包含两种架构：

#### CBOW（Continuous Bag of Words）
根据上下文词预测中心词。
```
上下文：["我", "喜欢", "小", "动物"]
预测：["猫"]
```

#### Skip-gram
根据中心词预测上下文词。
```
输入：["猫"]
预测：["我", "喜欢", "小", "动物"]
```

**特点：**
- 训练速度快
- 能捕捉语义关系
- 向量维度：通常 100-300 维

**有趣的性质：**
```
vec("国王") - vec("男人") + vec("女人") ≈ vec("女王")
vec("北京") - vec("中国") + vec("日本") ≈ vec("东京")
```

### GloVe（Global Vectors）
斯坦福提出的词嵌入方法，结合全局统计信息和局部上下文信息。

**核心思想：**
词共现矩阵中的比值可以编码语义关系。

**优势：**
- 利用全局统计信息
- 训练速度较快
- 在类比任务上表现优异

### FastText（2016）
Facebook 提出的词嵌入方法，主要改进：

#### 子词信息（Subword Information）
将词表示为字符 n-gram 的组合：
```
"apple" → {"<ap", "app", "ppl", "ple", "le>"}
```

**优势：**
- 处理未登录词（OOV）
- 捕捉词形变化规律
- 支持多种语言

## 上下文相关词嵌入

### 问题
传统词嵌入是静态的，每个词只有一个向量表示，无法处理一词多义：
- "bank"（银行）vs "bank"（河岸）
- "苹果"（水果）vs "苹果"（公司）

### ELMo（Embeddings from Language Models）
基于双向 LSTM 的语言模型，生成上下文相关的词表示。

**特点：**
- 深层双向表示
- 动态词嵌入
- 可迁移到下游任务

### BERT 嵌入
BERT 生成的上下文相关嵌入，在不同层捕捉不同级别的语义信息：
- 底层：词法、句法信息
- 中层：语义信息
- 高层：任务相关特征

## 句子嵌入与文档嵌入

### 简单方法
- **平均池化**：取词向量的平均值
- **最大池化**：取每个维度的最大值

### 高级方法

#### Sentence-BERT（SBERT）
使用孪生网络结构学习句子嵌入，适用于语义相似度计算。

#### Universal Sentence Encoder
Google 提出的通用句子编码器，支持多语言。

#### Doc2Vec
扩展 Word2Vec 到文档级别：
- **PV-DM**：类似 CBOW，加入文档向量
- **PV-DBOW**：类似 Skip-gram

## 评估词嵌入

### 内在评估
- **词类比任务**：king - man + woman = queen
- **词相似度**：与人工标注的相关性对比
- **词语义相关性**：判断词对的相关程度

### 外在评估
将词嵌入应用于下游任务，评估任务性能提升：
- 文本分类
- 命名实体识别
- 情感分析

## 应用实践

### 使用预训练词向量
```python
import gensim.downloader as api

# 加载预训练词向量
word_vectors = api.load("word2vec-google-news-300")

# 获取词向量
vector = word_vectors["computer"]

# 找相似词
similar_words = word_vectors.most_similar("computer", topn=5)
```

### 训练自定义词向量
```python
from gensim.models import Word2Vec

# 准备语料（分词后的句子列表）
sentences = [["我", "喜欢", "自然", "语言", "处理"], ...]

# 训练模型
model = Word2Vec(sentences, vector_size=100, window=5, min_count=1)

# 保存模型
model.save("word2vec.model")
```

## 总结

词嵌入技术的发展历程：
1. **One-Hot** → 稀疏、高维、无语义
2. **Word2Vec/GloVe** → 密集、低维、静态语义
3. **ELMo/BERT** → 上下文相关、动态语义
4. **Sentence/Doc Embedding** → 句子/文档级别表示

选择合适的词嵌入方法需要考虑：任务类型、数据规模、计算资源、是否需要处理一词多义等因素。
